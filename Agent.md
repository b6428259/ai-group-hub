# เอเจนต์แฮนด์บุ๊ค (Agent.md) — ระบบควบคุมกลุ่ม AI (AI Group Hub)

เอกสารนี้รวบรวมรายละเอียดสถาปัตยกรรมระบบ, โครงสร้างการทำงาน, งานที่ทำเสร็จแล้ว, สิ่งที่ต้องดำเนินการต่อ และข้อควรระวังสำคัญ สำหรับนักพัฒนาหรือเอเจนต์ที่จะมารับช่วงงานต่อไป

---

## 🏗️ 1. สถาปัตยกรรมระบบ (System Architecture)

ระบบได้รับการปรับโครงสร้างใหม่เพื่อแยกส่วนหน้าบ้านและหลังบ้านออกจากกันอย่างเด็ดขาด (Decoupled Architecture) เพื่อเตรียมพร้อมสำหรับจำลองใช้งานออนไลน์จริง:

1.  **หน้าบ้าน (Frontend App)**: เป็นเว็บ React/Vite แบบ Static Application ซึ่งถูกคอมไพล์และ Deploy ขึ้นระบบ **Vercel**
2.  **หลังบ้าน (Standalone Backend)**: เป็น Express API Server ขนาดเล็ก รันอยู่บนเครื่อง **ION Server (`100.96.8.110`)** คอยทำหน้าที่รับคำขอ ส่งคำขอ และดึงข้อมูลจาก OpenClaw CLI

## ⚙️ 2. กลไกการทำงานหลัก (How It Works)

### A. โครงสร้างองค์กรแบบบริษัทใหญ่ (Corporate Swarm Hierarchy)
ระบบควบคุมกลุ่ม AI จำลองระบบตามสายงานองค์กรแบบพรีเมียม ประกอบด้วยเอเจนต์ 7 ตัวหลัก:
*   **Secretary (เลขาธิการ)**: รับโจทย์งานจากผู้ใช้ จัดข้อประชุม และเสนอต่อ CEO
*   **CEO (ผู้บริหารสูงสุด)**: กำหนดทิศทาง วางเป้าหมาย และส่งมอบเป้าหมายต่อให้ผู้อำนวยการ
*   **CTO (ผู้อำนวยการฝ่ายเทคโนโลยี)**: ออกแบบโครงสร้างระบบ โครงสร้างไฟล์ และโมดูลที่ใช้
*   **PM (ผู้จัดการผลิตภัณฑ์)**: นำโครงสร้างจาก CTO มาแตกงานเป็นชิ้นงานย่อย (Sub-tasks) เพื่อจ่ายให้ลูกน้อง
*   **Developer (โปรแกรมเมอร์)**: เขียนโค้ด พัฒนาสคริปต์
*   **Copywriter (นักเขียน)**: เขียนเนื้อหา เอกสารประกอบ หรือสูตรอาหาร
*   **QA Inspector (ผู้ตรวจสอบคุณภาพ)**: ตรวจโค้ดและเนื้อหาหาจุดผิดพลาด

### B. ระบบรันงานขนาน (DAG Parallel Scheduler)
แทนที่จะทำงานแบบเรียงแถว 1-2-3 ทีละตัว ระบบนี้ใช้ **Directed Acyclic Graph (DAG)** เพื่อประเมินความเชื่อมโยงของแต่ละขั้นตอน:
*   ขั้นตอนที่ไม่มีความเชื่อมโยงกัน (เช่น งานวิจัยเบื้องต้น หรือขั้นตอนที่จ่ายงานให้ Developer กับ Copywriter ทำคู่กัน) **จะถูกประมวลผลไปพร้อมกันทันทีในลักษณะ Parallel**
*   ช่วยลดระยะเวลารอคอยคำตอบจาก AI ในภาพรวมลงได้เกือบ 50%

### C. การเชื่อมต่อ SUT HTTP ตรง (Direct API Bypass)
*   เนื่องจากระบบ Windows มีข้อจำกัดความยาวของอาร์กิวเมนต์หน้าต่างเชลล์อยู่ที่ 32KB (`The command line is too long`) เมื่อนำผลลัพธ์ของแต่ละขั้นมารวมกัน จึงอาจเกิดการแครช
*   **การแก้ไข**: ระบบหลังบ้านจะละทิ้งการส่งคำสั่งผ่าน CLI สำหรับขั้นตอน Inference แต่จะทำการเปิด HTTP Request ตรงไปหา **SUT OpenWebUI API (`https://genai.sut.ac.th/api/chat/completions`)** ผ่าน Fetch ทำให้หมดกังวลเรื่องความยาวประโยคคำสั่ง และประมวลผลเร็วขึ้นมาก

### D. ระบบถอดรหัส 3 ชั้น (3-Tier Planner Parser)
โมดูลแผนงานของ CEO สามารถดึงข้อมูลขั้นตอนและเอเจนต์ได้อย่างปลอดภัยผ่าน 3 ด่าน:
1.  **JSON Parser**: พยายามแกะอาร์เรย์มาตรฐาน
2.  **XML Parser**: พยายามแกะเครื่องหมาย `<step>` (ทนทานต่ออักขระพิเศษและเครื่องหมายคำพูด)
3.  **Markdown Parser**: ใช้ Regular Expressions ในการตัดแบ่งหัวข้อ Markdown (ทนทานต่อนโยบายของระบบ SUT ที่มักบังคับบอทให้เกริ่นนำหรือตอบเป็นหัวข้อลิสต์)

### E. ระบบพรีวิวแอพสด (Live App Preview Sandbox)
*   หากผลงานที่ส่งมอบมีไฟล์ประเภท HTML (`index.html`) จะมีปุ่ม **Preview** แสดงขึ้นมาบนการ์ดผลงาน
*   เมื่อคลิก จะรันแอพนั้นในกล่อง `iframe` บนหน้าเว็บทันทีแบบ Sandboxed ช่วยให้คุณลองใช้แอพหรือเล่นเกมที่บอทพึ่งเขียนเสร็จได้ทันที

## ⚠️ 5. ข้อควรจำและข้อควรระวัง (Things to Know)

*   **ปัญหา CORS (Cross-Origin Resource Sharing)**: หลังบ้าน Express จำเป็นต้องเปิดใช้งานมิดเดิลแวร์ `cors` เสมอ เพื่ออนุญาตให้โดเมน Vercel ส่งคำขอข้ามโดเมนเข้ามาประมวลผลได้
*   **การเชื่อมต่อ OpenClaw CLI**: เมธอด `GET /api/models` ยังจำเป็นต้องใช้การเรียกคำสั่ง `openclaw models list --json` ดังนั้นเครื่อง ION Server จำเป็นต้องติดตั้ง OpenClaw และมีการล็อกอินบัญชีเรียบร้อยแล้ว
*   **API Key**: ตัวระบบหลังบ้านต้องการ `SUT_OPENWEBUI_API_KEY` เสมอ โดยจะสแกนหาจาก Environment Variable หรือไฟล์เครื่องส่วนตัว `C:\Users\iooon\.openclaw\agents\main\agent\models.json` บนบู๊ตเครื่อง
